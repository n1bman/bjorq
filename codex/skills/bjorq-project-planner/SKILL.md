# BJORQ Project Planner

Keep planning short, concrete, and operational.

## Main Files

- `codex/planning/MASTER-PLAN.md`
- `codex/planning/BACKLOG.md`
- `codex/planning/ISSUES-AND-FIXES.md`

## Update Flow

1. Put current priorities and operating rules in `MASTER-PLAN.md`.
2. Put candidate work and ideas in `BACKLOG.md`.
3. Put verified bugs, risks, and completed fixes in `ISSUES-AND-FIXES.md`.

## Rules

- Separate confirmed problems from assumptions.
- Prefer one clear line item over long narrative notes.
- Keep product direction anchored in BJORQ as a 3D smart home dashboard across web, tablet, and phone.
